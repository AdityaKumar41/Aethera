Aethera – FULL FUND FLOW (START TO END)

Think of Aethera as infrastructure financing, not crypto.

0️⃣ Who puts money? Who takes money?

Let’s lock roles first.

Installer / Developer

Needs money

Uses money to build solar project

Generates electricity

Sells electricity

Pays back investors indirectly via revenue

Investor

Puts money

Does NOT get principal back like a loan

Gets periodic yield from revenue

Optionally exits by selling tokens

Aethera Platform

Does NOT create money

Routes money

Enforces rules

Takes small fees

1️⃣ HOW INSTALLER RAISES MONEY (MOST IMPORTANT PART)
Question you asked:

“On what basis does an installer raise money?”

Answer:

On the basis of future cashflows from electricity generation.

This is exactly how solar is financed today, just with better rails.

1.1 Installer submits a project

Installer provides:

Capacity (e.g. 500 kW)

Expected annual generation (kWh)

Electricity selling price (PPA / tariff)

Project cost (CAPEX)

Timeline

Example:

Capacity: 500 kW

Annual generation: 750,000 kWh

Tariff: ₹6 / kWh

Annual revenue: ₹45,00,000

This revenue forecast is the basis for fundraising.

1.2 Platform evaluates the project

Aethera checks:

Is generation realistic?

Is tariff contract valid?

Is cost reasonable?

Is location suitable?

If approved → project is allowed to raise funds.

1.3 Project is tokenized (KEY STEP)

Here is where tokenization starts to matter.

What token represents

A fractional claim on future project revenue

Not:

❌ A loan

❌ Equity shares

❌ A promise of fixed return

Instead:

“If this project earns money, you earn your share.”

1.4 Funding target is set

Example:

Project cost: ₹3 crore

Funding target: ₹3 crore

Token supply: 3,000 tokens

Price per token: ₹10,000

Each token ≈ 0.033% of project revenue

2️⃣ INVESTOR PUTS MONEY (WHERE DOES IT GO?)
Investor invests ₹10,000 × N tokens
Money flow:
Investor → Treasury (escrow)


Important:

Money does NOT go to installer immediately

It stays locked until conditions are met

2.1 What investor actually gets

Investor gets:

Tokens in their wallet

Tokens represent revenue rights

Tokens are backed by:

Real project

Legal agreements

On-chain rules

This is the receipt for their money.

3️⃣ WHEN DOES INSTALLER GET THE MONEY?

This is crucial.

Condition:

Only when funding target is met (or milestone-based).

Then:
Treasury → Installer


Now installer has capital to:

Buy panels

Install infrastructure

Connect to grid

This protects investors from:

Half-funded projects

Money misuse

4️⃣ HOW DOES MONEY COME BACK TO INVESTORS?

This is where confusion usually happens.

IMPORTANT:

Investors are NOT repaid their principal like a loan.

Instead:

Investors receive yield from revenue

Over time, yield may exceed initial investment

4.1 Electricity revenue flow

Once solar plant is live:

Solar plant
   ↓
Electricity sold
   ↓
Utility / Offtaker pays installer


This happens off-chain, in the real world.

4.2 Revenue enters the Aethera system

Installer is required (contractually) to:

Periodically send revenue to platform treasury

Or give platform revenue visibility

Example:

Monthly revenue: ₹3,75,000

Platform fee: ₹25,000

Distributable: ₹3,50,000

4.3 Yield distribution to investors

Treasury distributes money:

Treasury → Investors (pro-rata)


If investor owns 1% of tokens:

They get 1% of ₹3,50,000 = ₹3,500

This repeats every month / quarter.

5️⃣ WHAT ABOUT PRINCIPAL? (VERY IMPORTANT)

There are 3 valid models. You choose one.

Model A – Revenue-share (most honest)

No principal repayment

Investor earns yield forever (or project lifetime)

Token value depends on expected future cashflows

Used in:

Infrastructure funds

Royalties

Streaming deals

Model B – Revenue-share with buyback

Part of revenue used to:

Buy back tokens

Burn them

Eventually investor recovers principal + profit

This feels safer to retail investors.

Model C – Hybrid (most common)

Yield paid regularly

After X years:

Installer buys back tokens

Or project is refinanced

This matches traditional solar finance.

6️⃣ WHAT IF INVESTOR WANTS EXIT EARLY?

Investor has options:

Option 1 – Sell tokens

To another investor

On platform-approved market

Option 2 – Hold until buyback

If project has buyback clause

Option 3 – Governance-approved exit

Project-level decision

7️⃣ WHAT IF PROJECT FAILS?

This is REAL LIFE — must be clear.

If production is low:

Yield is low

If project shuts down:

Yield stops

Investor risk:

Infrastructure risk

No fake guarantees

Tokenization does NOT remove risk.
It makes risk visible and priced.
